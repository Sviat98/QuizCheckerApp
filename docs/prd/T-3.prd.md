# T-3: Страница добавления нового бланка в HorizontalPager

Status: DRAFT

## Контекст / идея

На экране просмотра бланков (вкладка "Blanks") используется `HorizontalPager` с бесконечной прокруткой по списку `blankTemplates`. Нужно добавить в конец каждого цикла пейджера дополнительную страницу — "Добавить новый бланк". На этой странице должна быть кнопка, при нажатии на которую выезжает текстовое поле (textarea) для ввода plain text и кнопка "Добавить". Пока что отправка данных на сервер не требуется — только UI.

## Цели

1. Добавить специальную страницу "Добавить новый бланк" в `HorizontalPager` после всех blankTemplates
2. Изменить логику бесконечной прокрутки: цикл теперь состоит из `blankTemplates.size + 1` страниц
3. На странице добавления отобразить кнопку "Добавить новый бланк"
4. По нажатию на кнопку — анимированно выезжает textarea + кнопка "Добавить"
5. Кнопка "Добавить" пока ничего не делает (поведение будет добавлено позже)

## User stories

- Как администратор, я хочу свайпнуть до последней страницы в пейджере бланков и увидеть экран добавления нового бланка
- Как администратор, я хочу нажать кнопку "Добавить новый бланк" и увидеть текстовое поле для ввода
- Как администратор, я хочу ввести текст нового бланка в textarea

## Основные сценарии

### Сценарий 1: Отображение страницы добавления
1. Пользователь свайпает вправо по пейджеру бланков
2. После последнего blankTemplate отображается страница "Добавить новый бланк"
3. На странице видна кнопка "Добавить новый бланк"
4. Страница стилизована в соответствии с общим дизайном экрана бланков

### Сценарий 2: Раскрытие формы ввода
1. Пользователь нажимает кнопку "Добавить новый бланк"
2. Текстовое поле (textarea) выезжает снизу с анимацией
3. Под textarea появляется кнопка "Добавить"
4. Пользователь может ввести текст в textarea

### Сценарий 3: Нет blankTemplates
1. Если `blankTemplates` пуст — вместо пустого экрана "No blank templates" отображается пейджер с одной страницей — "Добавить новый бланк"

## Успех / метрики

- Страница добавления корректно отображается как последняя в цикле пейджера
- Анимация выезжания textarea плавная
- Бесконечная прокрутка работает корректно с новой длиной цикла

## Ограничения и допущения

- Формат ввода — plain text (произвольный текст)
- Кнопка "Добавить" пока не отправляет данные на сервер — только UI
- Текущая бесконечная прокрутка (модульное индексирование через `Int.MAX_VALUE`) сохраняется, но длина цикла увеличивается на 1
- Сворачивание textarea (обратная анимация) не требуется в рамках этой задачи

## Техническая реализация

### Изменения в presentation layer:

1. **`QuizDetailsBlanksState.kt`** — добавить состояние:
   - `isAddBlankFormVisible: Boolean = false` — видимость формы ввода нового бланка

2. **`QuizDetailsBlanksEvent.kt`** — добавить события:
   - `data object ShowAddBlankForm : QuizDetailsBlanksEvent` — раскрыть форму добавления

3. **`QuizDetailsBlanksViewModel.kt`** — добавить обработку:
   - В `onEvent`: при `ShowAddBlankForm` — установить `isAddBlankFormVisible = true`

4. **`QuizDetailsBlanksScreen.kt`** — основные изменения:
   - **`BlankTemplatesPager`**: изменить `pageCount` с `Int.MAX_VALUE` на `Int.MAX_VALUE`, но цикл = `blankTemplates.size + 1`
     - Для `pageIndex % cycleSize < blankTemplates.size` — отобразить `BlankTemplatePage`
     - Для `pageIndex % cycleSize == blankTemplates.size` — отобразить `AddBlankTemplatePage`
   - **`QuizDetailsBlanksScreen`**: убрать ветку `state.blankTemplates.isEmpty()` — вместо неё пейджер с одной страницей "Добавить"
   - Новый composable **`AddBlankTemplatePage`**:
     - Принимает `isFormVisible: Boolean`, `onShowForm: () -> Unit`
     - Отображает кнопку "Добавить новый бланк" по центру экрана
     - При нажатии — вызывает `onShowForm`
     - Если `isFormVisible` — отображает textarea + кнопку "Добавить" (с анимацией выезжания)
     - Textarea: `TextField` с `maxLines = Int.MAX_VALUE` (или `OutlinedTextField`), `Modifier.fillMaxWidth()`
     - Кнопка "Добавить" — пока без обработчика (заглушка)
