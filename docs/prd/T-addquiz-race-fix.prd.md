# T-addquiz-race-fix: Исправление эффекта гонки при добавлении квиза

Status: DRAFT

## Контекст / идея

При нажатии кнопки "Добавить квиз" в диалоге `AddQuizScreen` одновременно происходят два действия: запуск сетевого запроса (`onEvent(AddQuiz)`) и немедленная навигация назад (`onBack()`). Это приводит к тому, что ViewModel уничтожается до завершения корутины, выбрасывается `CancellationException`, хотя сам сетевой запрос успевает выполниться на сервере.

## Цели

1. Устранить `CancellationException` при добавлении квиза.
2. Выполнять навигацию назад автоматически только после успешного завершения сетевого запроса.
3. Не вводить новый action — использовать реакцию на изменение состояния экрана.

## User stories

- Как администратор, я хочу, чтобы после нажатия "Добавить квиз" диалог закрывался только после успешного сохранения квиза на сервере, чтобы я был уверен, что операция выполнена.

## Основные сценарии

### Успешное добавление
1. Администратор заполняет форму и нажимает "Добавить квиз".
2. UI отправляет событие `AddQuiz`.
3. ViewModel запускает сетевой запрос в `viewModelScope`.
4. Сервер возвращает `Success`.
5. ViewModel устанавливает `isAdded = true` в состоянии.
6. Экран наблюдает за `isAdded` и вызывает `navController.navigateUp()`.
7. Диалог закрывается.

### Ошибка сети
1. Администратор заполняет форму и нажимает "Добавить квиз".
2. UI отправляет событие `AddQuiz`.
3. ViewModel запускает сетевой запрос в `viewModelScope`.
4. Сервер возвращает ошибку.
5. `isAdded` остаётся `false`.
6. Диалог остаётся открытым, пользователь может повторить попытку.

## Успех / метрики

- Отсутствие `CancellationException` в логах при добавлении квиза.
- Диалог не закрывается, пока сервер не подтвердил успешное создание.

## Ограничения и допущения

- `AddQuizViewModel` зарегистрирован через `factoryOf` в Koin, поэтому при каждом навигационном переходе на `AddQuizRoute` создаётся новый экземпляр с `isAdded = false` из `initial()`. Явный сброс `isAdded` не требуется.
- `QuizRepository.addQuiz()` возвращает `LoadResult`, но текущая реализация `QuizRepositoryImpl.addQuiz()` не возвращает его. Метод нужно изменить, чтобы он возвращал `LoadResult<Quiz, NetworkError>` для обработки результата во ViewModel.
