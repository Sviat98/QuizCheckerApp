# T-7: Линейка (Divider) под ответом в слоте бланка

Status: DRAFT

## Контекст / идея

В отрисовке слота бланка на экране шаблонов (`QuizDetailsBlanksScreen`) отсутствует визуальный разделитель (линейка) под ответом и баллами. Необходимо добавить горизонтальную линию (Divider) снизу каждого слота для улучшения визуального оформления и структурированности бланка.

## Цели

- Добавить горизонтальную линейку (Divider) под ответом и баллами в каждом слоте бланка
- Номер слота должен быть визуально чуть крупнее, чтобы его высота была равна высоте строки ответ + линейка
- При множественных ответах — линейка есть всегда, но ТОЛЬКО под ответом (не под иконкой)
- При раскрытии ответов — линейка остаётся одна внизу

## User stories

- Как администратор, я хочу видеть линейку под каждым ответом в слоте, чтобы бланк выглядел структурированно и аккуратно
- Как администратор, я хочу чтобы при раскрытии нескольких ответов линейка была одна — в самом низу, чтобы не перегружать интерфейс

## Основные сценарии

### Сценарий 1: Слот с одним ответом
```
[slotNumber.]  [answer text]  [points]
─────────────────────────────────────
```
- Линейка всегда отображается (один ответ всегда есть)
- Номер слота центрируется вертикально по высоте "ответ + линейка"
- Шрифт номера слота чуть крупнее (`titleSmall` вместо `bodyMedium`)

### Сценарий 2: Слот с несколькими ответами, свёрнутое состояние
```
[slotNumber.]  [answers label (3)]  [▼ icon]
               ─────────────────────
```
- Линейка есть, но ТОЛЬКО под текстом ответа (не доходит до иконки)
- Иконка раскрывающегося списка без линейки под ней

### Сценарий 3: Слот с несколькими ответами, раскрытое состояние
```
[slotNumber.]  [answers label (3)]  [▲ icon]
  answer 1                         points 1
  answer 2                         points 2
  answer 3                         points 3
─────────────────────────────────────
```
- Одна линейка внизу, после всех ответов (на всю ширину)
- При сворачивании линейка заменяется на линейку только под текстом (сценарий 2)

## Успех / метрики

- Визуальная гармония: номер слота, ответ и линейка выровнены
- Линейка присутствует только в нужных состояниях (один ответ / раскрытые ответы)
- Раскрытие/сворачивание работает плавно с анимацией

## Ограничения и допущения

- Линейка — простая горизонтальная линия (`HorizontalDivider` из Material3)
- Слот с одним ответом всегда содержит ответ (по условию задачи)
- Линейка не должна перекрывать или визуально конфликтовать с другими элементами

## Риски

- Анимация раскрытия/сворачивания может некорректно работать с линейкой — нужно убедиться что линейка входит в `AnimatedVisibility`
